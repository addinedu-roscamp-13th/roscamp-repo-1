# 원격 농장 지휘 아키텍처 — 폴링 릴레이 (edge 아웃바운드)

## 요구사항 (이건수, 2026-07-10)
1. 라이브 웹앱을 농장 작업자들이 **전국·전세계 어디서든** 쉽게 접속
2. **원격 농장 로봇에게 명령** 전송
3. **농장으로부터 알람** 수신

## 핵심 문제 두 가지
1. 농장은 공개 주소가 없다(가정/농장 인터넷 = NAT 뒤) → 바깥에서 농장으로 "전화 걸기" 불가
2. 무료 pythonanywhere Web은 바깥으로 아웃바운드 호출 불가(외부연결 차단)
→ 둘 다 "클라우드가 농장으로 전화 거는" 방향이라 막힘.

## 해결: 방향을 뒤집는다 (카톡/IoT 방식)
농장(edge)이 공개 Web으로 **먼저·계속 연결(아웃바운드 폴링)**. Web은 명령을 큐에 쌓고,
농장이 가져가 실행 후 결과를 되쏜다. **Web은 아웃바운드 안 함 → 무료 호스팅 OK. 농장은 아웃바운드만 → NAT 무관.**

```
[브라우저: 전국/해외 어디서든]──▶[ Automato Web Service (공개 URL, 무료) ]◀── 폴링 ──[ 농장 ACS + 로봇 ]
     POST /patrol/requests          명령 큐에 적재                          POST /internal/v1/farm/poll (2s)
     GET  /patrol/events   ◀── 이벤트 푸시 ◀── 콜백 ◀────────────────────── POST detections/notify·alerts/disease·patrol/completed
     (텔레그램은 Web이 발송 — api.telegram.org는 무료 whitelist)
```

## 구현 (2026-07-10 배포 완료)
### Web(app.py) 추가 — 기존 기능 안 건드리고 얹기만
- `POST /internal/v1/farm/poll` : 농장이 주기 호출. 살아있음 표시 + 로봇상태 보고 + 대기명령 1건 수령.
- `GET /api/v1/farm/status` : 농장 연결상태(online/queued/last_poll).
- `_farm_online()` : 최근 15초 내 폴링 있으면 온라인.
- `patrol_available` : 농장 온라인이면 **농장이 보고한 실제 로봇상태** 반환.
- `patrol_request` : 농장 온라인이면 **명령 큐 적재**(농장이 실행). 없으면 기존 셀프시뮬(데모).
- 우선순위: `CONTROL_SERVICE_URL 직결(중계)` > `농장 폴링(큐)` > `셀프시뮬(데모)`.

### 농장 에이전트(farm_agent.py) — edge 측 (보연님 실 ACS가 이 자리)
- `WEB_SERVICE_URL` 로 2초 주기 아웃바운드 폴링. 명령 받으면 순찰 실행 → 콜백 POST.
- 실행: `WEB_SERVICE_URL=https://geonsulee.pythonanywhere.com python3 farm_agent.py`

## 검증 (둘 다 통과)
- 로컬: Web+farm_agent → 명령 큐→폴링수령→실행→콜백→이벤트/텔레그램 ✅
- **공개 URL**: 내 PC(NAT 뒤)의 farm_agent가 `https://geonsulee.pythonanywhere.com` 로 아웃바운드 폴링 →
  공개 URL에 낸 순찰명령을 받아 실행 → 브라우저 이벤트(wp0~3, 병해충7%, 완료) + 텔레그램 ✅
  → **포트개방·공개IP 없이 원격 지휘 성립.**

## 요구사항 충족
| 요구 | 충족 | 방법 |
|---|---|---|
| ① 전국/해외 접속 | ✅ | 공개 URL 그대로(무료 유지) |
| ② 원격 로봇 명령 | ✅ | 브라우저→Web 큐→농장 폴링 수령→실행 |
| ③ 농장 알람 수신 | ✅ | 농장→Web 콜백→브라우저 이벤트 + 텔레그램(폰 어디서든) |

## 내일(실기) 전환
보연님 실제 ACS가 **폴링 규약**(`POST /internal/v1/farm/poll` 로 farm_id+robots 보내고 command 받기)만
따르면 farm_agent 자리에 그대로 들어옴. 콜백 3종(detections/notify·alerts/disease·patrol/completed)은 스펙 그대로.
※ 이는 "Web→ACS 직결(POST /internal/v1/tasks/patrol)" 스펙을 **원격 농장용으로 확장**한 것 — 팀/보연님 공유 필요.

## 알림 정책(스펙 고정)
텔레그램·브라우저 알림은 **오직 disease_alert(E3) + patrol_completed(E2-10)** 에서만. 그 외 없음.
