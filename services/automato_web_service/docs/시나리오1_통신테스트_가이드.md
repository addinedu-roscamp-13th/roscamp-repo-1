# 시나리오1 통신테스트 — 순서대로 "어떤 버튼을 누르면 되는지"
### Farm Admin App ↔ Automato Web Service(나) ↔ Automato Control Service(보연님 = 가상 ACS)

> 목표: 시나리오1(컨플루언스 23691289)의 **모든 경우의 수**를 순서대로 직접 눌러서,
> 요청이 **가상 ACS까지 실제 도달**하는 걸 3곳(**왼쪽 터미널=ACS 로그 · 웹앱 화면 · Postman**)에서 내 눈으로 확인한다.
> 반영 기준: 2026-07-08 회의(disease≥5·dedup 없음·사진 첨부·저장은 ACS) + 2026-07-08 필드 퍼센트화(ripe/unripe/rotten/disease_percent, total_count 삭제) — **전부 반영 완료**.

## 0. 준비 — 터미널 2개 + 브라우저 + (선택)Postman

**왼쪽 터미널 = 가상 ACS(보연님 대역).** 여기 로그(`[가상-ACS] ◀ Web: …`)가 통신 도달 증거다. 이 창을 계속 본다.
```bash
cd ~/Desktop/claude/automato-live && WEB_SERVICE_URL=http://127.0.0.1:8899 PORT=7001 python3 mock_control_service.py
```

**오른쪽 터미널 = 내 Automato Web Service.** 여기도 읽기 쉬운 로그(`[Web] ▶ ACS 중계 / ◀ ACS 콜백 → App 푸시`)가 흐른다.
```bash
cd ~/Desktop/claude/automato-live && CONTROL_SERVICE_URL=http://127.0.0.1:7001 PORT=8899 python3 app.py
```

**브라우저**: ⚠️ **`http://localhost:8899`** 로 연다. (pythonanywhere 아님! 그건 라이브 배포판이라 로컬 ACS에 안 붙는다.)
→ 자동으로 관리자 앱 진입 → 첫 로딩 시 **알림 권한 "허용"** 클릭.

**(선택) Postman**: `docs/Automato_WebService.postman_collection.json` Import → 변수 `base = http://localhost:8899`.
→ 웹앱 UI로 못 누르는 케이스(거절 2종·수동 콜백)는 Postman으로 쏜다.

> 버튼 한 번의 흐름: **웹앱 → [오른쪽 터미널 Web] ▶ 중계 → [왼쪽 터미널 ACS] 수신·콜백 → [오른쪽 터미널 Web] ◀ App 푸시 → 웹앱 화면**.
> 매 단계 **① 오른쪽(Web) ② 왼쪽(ACS) ③ 웹앱 화면** 세 곳이 같이 움직이는 게 3자 통신 증거.
>
> **알림 정책(스펙 고정)**: 텔레그램·브라우저 알림은 **오직 병해충(disease_alert, E3) + 순찰완료(patrol_completed, E2-10) 두 이벤트에서만** 발송. waypoint 진행·수확·기타 동작은 알림 안 감.

## 1. E1 순찰 시작 — 모든 경우의 수

### ▶ STEP 1. 순찰 가능 로봇 조회 (3상태) — E1-0/1
- **버튼**: 농장 홈 → **`🚓 순찰 보내기`** 타일 클릭
- **웹앱**: 카드 3개 — **DG1 배정 가능** / **DG2 🔋 배터리 부족** / **DG3 🚧 순찰 중**
- **왼쪽 터미널(ACS)**: `◀ Web: GET available → [('dg_01','가능'),('dg_02','BATTERY_TOO_LOW'),('dg_03','ROBOT_BUSY')]`
- ✅ available 조회도 Web이 ACS로 중계 → **App→Web→ACS 도달 확인**

### ▶ STEP 2. 자동 배정 요청 (auto → ACCEPTED) — E1-2/3
- **버튼**: **`⚡ 자동 배정`** → **`🚓 순찰 요청`**
- **웹앱**: `✅ 순찰 요청이 접수되었습니다 · 배정 로봇 dg_01 · #task_id`
- **왼쪽 터미널(ACS)**: `◀ POST tasks/patrol robot_selection=auto` → `→ 접수: task_id=… robot=dg_01 (상태 PATROLLING)`
- ⚠️ **이 한 번으로 아래 STEP 3(E2·E3·완료)이 자동으로 이어짐** — 왼쪽 터미널과 웹앱 상단을 계속 본다.

---

## 2. E2 웨이포인트별 검출 + E3 병해충 (STEP 2 요청 후 자동 관찰)

가상 ACS가 waypoint 0→1→2→3 을 0.8초 간격으로 돌며 자동 콜백한다.

### ▶ STEP 3-A. 검출 현황 실시간 (E2, 퍼센트)
- **웹앱 상단(`순찰현황`)**: `🚓 순찰 진행 중 · WP0…WP3 · 익음/안익음/썩음/병해충 %` 실시간 갱신
- **왼쪽 터미널(ACS)**: `▶ Web: notify wp0 … wp1 … wp2 … wp3 …`
- **(Postman)** `3. GET 이벤트 폴링` → `patrol_progress`(ripe/unripe/rotten/disease_percent) 확인

### ▶ STEP 3-B. disease < 5 → 알림 스킵 (E2/E3 경계)
- wp2 = `disease_percent 4%` → **병해충 배너 안 뜸(정상)**
- **왼쪽 터미널(ACS)**: `▶ Web: notify wp2 disease=4%  (5%미만 스킵)`

### ▶ STEP 3-C. disease ≥ 5 → 병해충 알림 (E3)
- wp3 = `disease_percent 7%` → **발동**
- **왼쪽 터미널(ACS)**: `▶ Web: notify wp3 disease=7%  → 병해충 알림 발동`
- **웹앱**: 상단에 **빨간 병해충 배너**(dg_01 · WP3 · 7% + 사진 썸네일) + 브라우저 알림 + 텔레그램 발송
- **(Postman)** 이벤트 폴링에 `disease_alert`(image_path 포함)
- ※ 지금은 로봇 OFF라 ACS 시뮬이 경로만 보냄 → 배너 썸네일은 🐛 대체 아이콘. **실제 레이블 사진은 내일 보연님 실 ACS가 image_data(base64)로 실어 보내면 그대로 뜸.**

### ▶ STEP 3-D. 완료 (patrol_completed)
- **왼쪽 터미널(ACS)**: `▶ Web: patrol_completed task_id=… (dg_01 대기복귀)`
- **웹앱**: 순찰현황 → "진행 중인 순찰이 없습니다"로 초기화, DG1 다시 **배정 가능**

## 3. E1 나머지 경우의 수 (완료로 dg_01 IDLE 복귀 후)

### ▶ STEP 4. 직접 선택 요청 (specific → ACCEPTED) — E1-2/3
- **버튼**: **`👆 직접 선택`** → **DG1 카드 클릭**(테두리 선택됨) → **`🚓 순찰 요청 · dg_01`**
- **웹앱**: `✅ … 배정 로봇 dg_01`
- **왼쪽 터미널(ACS)**: `◀ POST tasks/patrol robot_selection=specific robot_id=dg_01` → `→ 접수`
- (또 순찰이 돌므로, 완료될 때까지 ~4초 대기)

### ▶ STEP 5. 거절① 지정 로봇 불가 (ROBOT_NOT_AVAILABLE) — E1 409
> 웹앱은 불가 로봇(DG2) 카드를 클릭 자체가 막혀 있으므로 **Postman으로** 쏜다.
- **Postman**: `1. POST 순찰 요청` body를 `{"robot_selection":"specific","robot_id":"dg_02"}` 로
- **응답**: `409 REJECTED · reason: ROBOT_NOT_AVAILABLE`
- **왼쪽 터미널(ACS)**: `→ 거절: ROBOT_NOT_AVAILABLE (dg_02)`

### ▶ STEP 6. 거절② 가능 로봇 없음 (NO_AVAILABLE_ROBOT) — E1 409
- **웹앱 방법**: STEP 2로 dg_01을 순찰 내보낸 **직후(4초 안)** 순찰 보내기 화면을 새로 열면
  → **`🚫 지금 순찰 가능한 로봇이 없습니다`** 빨간 박스 → `🚫 순찰 요청 불가` 눌러 실제 409 확인
- **Postman 방법(확실)**: `1. POST 순찰 요청`(auto)을 **연속 2번**. 2번째 응답 `409 · NO_AVAILABLE_ROBOT`
- **왼쪽 터미널(ACS)**: `→ 거절: NO_AVAILABLE_ROBOT (가능 로봇 없음)`

## 4. (선택) 내가 직접 ACS 역할 — 수동 콜백 & 실사진 텔레그램

자동 흐름 말고 하나씩 직접 쏘고 싶으면 **Postman 폴더 2**:
- `2. POST 검출 현황 notify` → 웹앱 순찰현황 갱신
- `2. POST 병해충 알림 alerts/disease`(image_path=demo_label.jpg) → **웹앱 배너 + 텔레그램에 실제 레이블 샘플사진 도착**(로컬 detections/demo_label.jpg 존재)
- `2. POST 순찰 완료` → 완료 처리

## ✅ 체크리스트 (전부 왼쪽 터미널 ACS 로그로 도달 확인)
```
[ ] STEP1  available 3상태 (가능 / 배터리부족 / 순찰중)          ← ◀ GET available
[ ] STEP2  auto 배정 ACCEPTED                                   ← ◀ POST tasks/patrol auto → 접수
[ ] STEP3A E2 검출현황 퍼센트 실시간 (wp0~3)                    ← ▶ notify wp0..wp3
[ ] STEP3B disease<5 스킵 (wp2=4%, 배너 안 뜸)                  ← wp2 (5%미만 스킵)
[ ] STEP3C E3 disease≥5 병해충 배너 + 알림 + 텔레그램 (wp3=7%)  ← wp3 → 알림 발동
[ ] STEP3D 완료 patrol_completed (웹앱 초기화, dg_01 대기복귀)  ← patrol_completed
[ ] STEP4  specific 배정 ACCEPTED                               ← ◀ POST specific dg_01 → 접수
[ ] STEP5  거절 ROBOT_NOT_AVAILABLE (dg_02)                     ← → 거절 ROBOT_NOT_AVAILABLE
[ ] STEP6  거절 NO_AVAILABLE_ROBOT (가능 로봇 없음)             ← → 거절 NO_AVAILABLE_ROBOT
```

> **E0(상시 텔레메트리 루프)** 는 QT System Admin App ↔ ACS의 ROS2 직결이라 이 버튼 흐름(Farm Admin App 담당)에는 없음 — 시나리오1에서 내 파트는 E1·E2·E3.
> **내일 오후 3~4시 실통신**: 왼쪽 터미널 자리에 **보연님 진짜 ACS**를 놓고, 오른쪽 터미널의 `CONTROL_SERVICE_URL` 만 보연님 주소로 바꾸면 이 로그 자리에 진짜 ACS 응답이 찍힌다.
