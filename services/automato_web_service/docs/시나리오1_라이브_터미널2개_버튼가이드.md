# 시나리오1 — 라이브 웹앱 + 터미널 2개로 통신 보기
### 브라우저(라이브) 버튼 → [웹서비스 로그] + [컨트롤서비스] 두 터미널이 변한다

> 라이브 링크 `geonsulee.pythonanywhere.com` 에서 버튼을 누르면, 웹앱↔웹서비스↔컨트롤서비스 3층이 실제 통신.
> 웹서비스는 pythonanywhere(원격)에서 돌지만 **서버 로그를 API로 내 터미널에 실시간(약 2초 지연) tail** 해서 본다.
> 컨트롤서비스(farm_agent)는 내 PC에서 폴링 → **이 자리가 내일 보연님 실 ACS**. 오늘 되면 내일도 됨.

## 준비 — 터미널 2개 + 브라우저(라이브)

**터미널 ① = 웹서비스 로그 (라이브, 원격 → 내 화면)**
```bash
cd ~/Desktop/claude/automato-live && python3 web_log_tail.py
```
→ `[Web] …` 라인이 여기 흐름. (약 2초 지연)

**터미널 ② = 컨트롤서비스 (농장 = 내일 보연님 자리)**
```bash
cd ~/Desktop/claude/automato-live && WEB_SERVICE_URL=https://geonsulee.pythonanywhere.com FARM_ID=farm_gangneung_01 python3 farm_agent.py
```
→ 켜지면 몇 초 뒤 **터미널①에 `[Web] 🟢 원격 농장 접속`** 이 뜸(= 농장이 라이브에 붙음).

**브라우저**: `https://geonsulee.pythonanywhere.com` 새로고침 → 관리자 앱 → 알림 "허용".

---

## 버튼 순서대로 (시나리오1 전부)

### STEP 1. `🚓 순찰 보내기` 타일 — E1 available
- **브라우저**: DG1 가능(85%) · DG2 가능(74%) · DG3 🔋배터리부족(62%)
- **터미널①(웹)**: `[Web] ◀ App 요청: 순찰 가능 로봇 조회 → 농장 보고 상태 반환 [(dg_01,가능)(dg_02,가능)(dg_03,BATTERY_TOO_LOW)]`
- **터미널②(농장)**: 계속 폴링 중(명령 없으면 조용)

### STEP 2. `⚡ 자동 배정` → `🚓 순찰 요청` — E1 auto + E2 + E3 + 완료
- **브라우저**: `✅ 접수 · 농장에서 배정 중 · #task` → 순찰현황 WP0~3 → 병해충 배너 → 완료 + 텔레그램
- **터미널①(웹)**:
  ```
  [Web] ▶ App 요청: 순찰 요청(robot_selection=auto ...)
  [Web] ▶ 명령 큐 적재: patrol (task_id=..) — 원격 농장이 가져가길 대기
  [Web] ◀ 농장 폴링 → 명령 전달: patrol task_id=.. (농장이 실행)
  [Web] ◀ ACS 콜백: 검출현황 WP0..WP3 → App patrol_progress 푸시
  [Web] ◀ ACS 콜백: 병해충 알림 WP3 7% (≥5) → App disease_alert 푸시 + 텔레그램
  [Web] ◀ ACS 콜백: 순찰 완료 → App patrol_completed 푸시 + 텔레그램
  ```
- **터미널②(농장)**:
  ```
  [농장-ACS] ◀ Web에서 명령 수신: 순찰 task_id=..
  [농장-ACS]   ▶ 순찰 시작 robot=dg_01
  [농장-ACS]     ▶ Web: 검출 wp0..wp3 (wp3 7% → 병해충 알림)
  [농장-ACS]   ▶ Web: 순찰 완료 (dg_01 대기복귀)
  ```
> 흐름: 브라우저 → ①웹(명령 큐) → ②농장(폴링 수령·실행) → ①웹(콜백·앱푸시) → 브라우저.

### STEP 3. `👆 직접 선택` → **DG1 클릭** → `🚓 순찰 요청` — E1 specific
- STEP 2와 동일하되 터미널①에 `robot_selection=specific robot_id=dg_01`.

### STEP 4. 거절 — DG3(배터리부족) 지정 — E1 ROBOT_NOT_AVAILABLE
- **브라우저**: DG3 카드는 **클릭 자체가 막힘**(앱이 애초에 차단 = 정상). 원본 409 보려면 F12 콘솔:
  ```js
  fetch('/api/v1/patrol/requests',{method:'POST',headers:{'Content-Type':'application/json'},
    body:JSON.stringify({robot_selection:'specific',robot_id:'dg_03'})}).then(r=>r.json()).then(console.log)
  ```
- **터미널①(웹)**: `[Web] ▶ App 요청 … dg_03` → `→ 거절 ROBOT_NOT_AVAILABLE (dg_03 불가) → App 409`

### STEP 5. 거절 — 순찰 중 재요청 — E1 ALREADY_PATROLLING
- STEP 2로 순찰 내보낸 **직후(5초 안)** `🚓 순찰 보내기` 다시 열기 → 카드 전부 배정 불가 → `🚫 순찰 요청 불가` 클릭
- **터미널①(웹)**: `[Web] ▶ App 요청 …` → `→ 거절 ALREADY_PATROLLING (이미 순찰 중) → App 409`

## ✅ 체크리스트 (브라우저 + 터미널① + 터미널② 함께)
```
[ ] STEP1 available 3상태            ← 터미널① 조회 로그
[ ] STEP2 auto 수락 + E2/E3/완료      ← ①큐적재·콜백  ②명령수신·실행
[ ] STEP3 specific 수락               ← ① robot_id=dg_01
[ ] STEP4 거절 ROBOT_NOT_AVAILABLE    ← ① 거절 로그
[ ] STEP5 거절 ALREADY_PATROLLING     ← ① 거절 로그
```
알림(텔레그램/브라우저)은 **병해충(E3) + 순찰완료(E2-10)** 에서만. 그 외 없음.
