# 시나리오1 — 라이브(pythonanywhere)에서 버튼만 눌러 전부 통신
### 대상: https://geonsulee.pythonanywhere.com  (우리가 실제 쓰는 웹앱)

> 라이브 Web 서비스가 **외부 ACS 없이도 순찰 전 과정(E2 진행·E3 병해충·완료)을 스스로 진행**하도록 배포 완료(2026-07-09).
> 그래서 이 링크에서 **버튼만 눌러도** 시나리오1 흐름이 다 돈다. 통신 증거 = ①웹앱 화면 실시간 갱신 ②브라우저 알림 ③텔레그램 실제 발송.
> (로컬 터미널·가상 ACS는 여기서 필요 없음 — 그건 내일 보연님 실서버 붙일 때만.)

**준비**: 브라우저에서 `geonsulee.pythonanywhere.com` 열고 F5 → 관리자 앱 → 첫 로딩 시 **알림 "허용"**.

---

## STEP 1. 순찰 가능 로봇 조회 (E1-0/1) — 3상태
- **버튼**: 홈 → **`🚓 순찰 보내기`** 타일
- **화면**: 카드 3개 — **DG1 배정 가능(🔋85%)** / **DG2 배정 가능(🔋74%)** / **DG3 🔋 배터리 부족(62%<70)**
- ✅ Farm Admin App → 라이브 Web 실통신(GET available)

## STEP 2. 자동 배정 요청 (E1-2/3, auto → ACCEPTED)
- **버튼**: **`⚡ 자동 배정`** → **`🚓 순찰 요청`**
- **화면**: `✅ 순찰 요청이 접수되었습니다 · 배정 로봇 DG1 · #task` (배터리 가장 높은 DG1 자동 선정)
- ⚠️ **이 한 번으로 아래 STEP 3(E2·E3·완료)이 자동으로 흐름** — 약 5초간 화면을 본다.

## STEP 3. (자동 관찰) 웨이포인트 검출 → 병해충 → 완료
상단 **순찰현황** 카드를 본다. 약 1.2초 간격:
- **3-A (E2 검출)**: `🚓 순찰 중 · WP0 → WP1 → WP2 → WP3` + 익음/안익음/썩음/병해충 % 실시간 갱신
- **3-B (disease<5 스킵)**: **WP2 = 병해충 4%** → 배너 **안 뜸(정상)**
- **3-C (E3 병해충, disease≥5)**: **WP3 = 병해충 7%** → 🔴 **빨간 병해충 배너**(DG1·WP3·7%) + **브라우저 알림** + **텔레그램 발송**
- **3-D (완료)**: **"순찰 완료"** → 순찰현황 초기화, DG1 대기 복귀

## STEP 4. 직접 선택 요청 (E1-2/3, specific → ACCEPTED)
- (STEP 3 완료로 로봇 대기 복귀된 뒤)
- **버튼**: **`👆 직접 선택`** → **DG2 카드 클릭**(테두리 선택) → **`🚓 순찰 요청 · DG2`**
- **화면**: `✅ … 배정 로봇 DG2` → 다시 STEP 3 흐름 자동 진행

## STEP 5. 거절 — 순찰 중 재요청 (E1 409)
- **버튼**: STEP 2로 순찰 내보낸 **직후(5초 안)**, `🚓 순찰 보내기` 화면을 다시 열기
- **화면**: **`🚫 지금 순찰 가능한 로봇이 없습니다`** 빨간 박스 (순찰 중 1대 + 나머지 배터리부족)
  → `🚫 순찰 요청 불가` 누르면 **HTTP 409 · 요청 미접수** 빨간 응답 표시
- ✅ 거절 경로도 App↔Web 실통신으로 확인

## STEP 6. 거절 — 배터리 부족 로봇 (E1 ROBOT_NOT_AVAILABLE)
- **DG3(🔋배터리 부족)** 카드는 **직접 선택에서 클릭 자체가 막힘** = 앱이 애초에 거른다(정상 UX).
- 원본 409를 직접 보고 싶으면 브라우저 주소창(F12 콘솔)에서:
  ```js
  fetch('/api/v1/patrol/requests',{method:'POST',headers:{'Content-Type':'application/json'},
    body:JSON.stringify({robot_selection:'specific',robot_id:'DG3'})}).then(r=>r.json()).then(console.log)
  // → {status:'REJECTED', reason:'ROBOT_NOT_AVAILABLE', ...}
  ```

```
[ ] STEP3B disease<5 스킵 (WP2=4%, 배너 안 뜸)
[ ] STEP3C E3 병해충 배너 + 브라우저 알림 + 텔레그램 (WP3=7%)
[ ] STEP3D 완료 → 초기화, 로봇 대기 복귀
[ ] STEP4  specific 배정 ACCEPTED (DG2)
[ ] STEP5  거절 — 순찰 중 재요청 (가능 로봇 없음)
[ ] STEP6  거절 — 배터리부족 로봇 (앱이 차단 / 콘솔로 409 확인)
```

## 반영 확인 (슬랙 07/08 회의 + 컨플루언스 23691289)
- ✅ disease ≥ 5 만 알림 · 5 미만 스킵 (STEP 3B/3C)
- ✅ dedup 없음 (매 순찰마다 알림) · 사진 첨부 정책 · 저장은 ACS
- ✅ 필드 퍼센트화(ripe/unripe/rotten/disease_percent), total_count 삭제
- ⚠️ 병해충 **사진**만 로봇 OFF라 지금은 🐛 아이콘/텍스트. 실 레이블 사진 = 내일 보연님 ACS(D435→AI) 연동 시.
